# Kanken 在庫分配シミュレーター 運用マニュアル

## 【今回自動化・解決できること】
本システムは、毎月の最新の販売実績データを読み込ませるだけで、**「残り期間で何個売れるか（需要予測）」**と**「年末までに在庫を切らさないための店舗ごとの配分数」**を完全自動で計算し、ダッシュボード上に可視化します。
「1回きりの分析」ではなく、毎月実績を更新していくことで予測精度を高める「ローリングフォーキャスト」に対応しています。

---

## 【具体的な導入・運用ステップ】

毎月、新しい月の実績（例：6月の実績）が出た際は、以下の**3ステップ**だけでダッシュボードを最新状態に更新できます。

### ステップ1：実績データの準備と上書き
システムが入っているフォルダ（`シミュ２`フォルダ）にある、以下の**2つのファイル**の両方に、最新月の販売実績データを追記して上書き保存してください。

1. **`2026実績.csv`**
   - 新しい月の販売実績データを追加し、同じファイル名で上書き保存します。
2. **`分配店舗向け.xlsx`**
   - ファイルを開き、「2026売上」シートに新しい月の販売実績を追記して上書き保存します。
3. **`Kanken23510_2026年版_v5.xlsx`** （在庫や入荷予定が変わった場合のみ）
   - もし期首在庫や今後の入荷予定が更新された最新版のエクセルがあれば、同じファイル名でフォルダに上書き保存してください。（列名から日付を自動取得します）

### ステップ2：データ更新プログラムの実行
1. `シミュ２` フォルダの中にある **`データ更新を実行する.bat`** というファイルをダブルクリックします。
2. 黒い画面が立ち上がり、数秒〜数十秒で自動計算が行われます。
3. 「処理が完了しました！」と表示されたら、キーボードの何かキーを押して画面を閉じます。

### ステップ3：ダッシュボードの確認
1. 同じフォルダにある **`dashboard.html`** をダブルクリックしてブラウザで開きます。
2. （すでに開いている場合は、画面を「再読み込み（リロード）」してください）
3. 画面上部の「1-○月実績」や「○-12月売上予測」という文字が最新の月に切り替わり、すべてのシミュレーション数値が最新化されていることを確認してください。

## 【お使いいただく上での注意点（エラー回避策）】

システムを正常に動かすために、以下の点に十分ご注意ください。

- **🚨 必ず「Excel」と「CSV」の両方を更新してください**
  ステップ1で記載した `2026実績.csv` と `分配店舗向け.xlsx` のどちらか片方だけを更新してしまうと、システム内で「月の数」と「実際の売上」にズレが生じ、需要予測が異常な数字になってダッシュボードが崩壊する可能性があります。必ずセットで更新を行ってください。

- **🚨 列の名前（ヘッダー）を変更しないでください**
  各ファイル（ExcelやCSV）の1行目にある「カラー名」「数量」「MTH」などの列の名前を変更してしまうと、システムがデータを読み取れなくなります。

- **🚨 マイナスの実績が出た場合について**
  返品などで単月の販売実績がマイナスになった場合でも、システム側で自動的に「0」として足切りする安全装置が働いています。データ自体を修正していただく必要はありません。

- **🚨 12月までの実績が出揃った場合について**
  1年分の実績が全て揃った（予測する残り期間が0ヶ月になった）場合でも、エラーでシステムが停止することなく「売上予測（期間なし）」として表示されるよう安全対策が施されています。
